# Sprint 4 계획 — 게임 루프 완성 + 경제 활성화

## 목표
Sprint 1~3에서 시스템 골격(그리드, NPC, 엘리베이터, 렌더러, HUD, 빌드 모드)을 만들었고,
Sprint 4에서는 **실제로 게임을 플레이할 수 있는 수준**으로 끌어올린다.

핵심: "건물을 짓고 → 돈이 들어오고 → 별점이 변하고 → NPC가 반응하는" 루프가 돌아야 한다.

---

## 태스크 목록

### TASK-04-000: Design Spec v2.0 동기화 (⭐⭐)
- Sprint 3 전체 구현 반영 (HUDPanel, BuildCursor, FontManager, AnimationSystem 등)
- Sprint 4 경제 루프, 계단 이동, 메뉴 시스템 사양 추가
- DI-005 해소 방안 문서화

### TASK-04-001: 경제 루프 활성화 (⭐⭐⭐)
- Bootstrapper tick 루프에서 `collectRent()`, `payMaintenance()` 자동 호출
  - 매일(24틱) 임대료 수집
  - 매일 엘리베이터 유지비 차감
- `buildFloor()` 시 비용 차감 (`balance -= floorCost`)
  - 잔액 부족 시 거부 + HUD 피드백 (빨간색 메시지)
- `placeTenant()` 시 초기 인테리어 비용 차감
- HUD에 일일 수입/지출 표시 추가
- 테스트: 경제 시나리오 통합 테스트 (건설→임대→수입→잔액 변화)

### TASK-04-002: NPC 계단 이동 시스템 (⭐⭐⭐)
- `AgentSystem`에 계단 이동 로직 추가
  - 현재 층과 목표 층 차이 ≤ 4 → 계단 이동 (엘리베이터 안 탐)
  - 이동 속도: 엘리베이터보다 느림 (층당 2초)
  - 이동 중 상태: `Walking` (기존 스프라이트 재활용)
- NPC 경로 선택 로직: 계단 vs 엘리베이터 우선순위
  - 4층 이하 → 계단 우선
  - 5층 이상 → 엘리베이터 필수
  - 엘리베이터 대기 시간 > 임계값 → 4층 이하면 계단으로 전환
- 테스트: 계단 이동 단위 테스트 + 경로 선택 통합 테스트

### TASK-04-003: 정기 결산 시스템 (⭐⭐)
- `EconomyEngine::update()` 확장
  - 매일 자정: 일일 수입/지출 결산, 로그
  - 매주 일요일: 주간 리포트 이벤트 발생
  - 매분기 (90일): 분기 결산, 세금 차감, 별점 재평가
- 결산 이벤트 → HUD 알림 (ImGui 토스트)
- 테스트: 시간 경과 → 자동 결산 확인

### TASK-04-004: 게임 오버 + 승리 조건 (⭐⭐)
- 파산 조건: 잔액 < 0 상태 30일 지속 → 게임 오버
- TOWER 등급: ★5 도달 + 건물 100층 + NPC 300명 → TOWER 달성
- 게임 오버/TOWER 화면 (ImGui 모달)
- 테스트: 파산 시나리오, TOWER 달성 시나리오

### TASK-04-005: 메인 메뉴 + 새 게임/불러오기 (⭐⭐⭐)
- 게임 상태 머신: Menu → Playing → Paused → GameOver
- 메인 메뉴 (ImGui 전체 화면):
  - 새 게임 (setupInitialScene 기반)
  - 불러오기 (SaveLoadSystem 연결)
  - 설정 (해상도, 볼륨)
  - 종료
- ESC → 일시정지 메뉴 (계속/저장/메인으로)
- 테스트: 상태 전환 테스트

### TASK-04-006: DI-005 해결 + 테스트 품질 개선 (⭐⭐)
- MockGridSystem 엘리베이터 앵커 불일치 수정
- getTenantCount 단위 테스트 추가
- HUD 통합 테스트 (frame.balance ≠ 0 검증)
- 전체 Mock 일관성 감사

### TASK-04-007: 통합 + Sprint 4 빌드 (⭐⭐⭐⭐)
- 전체 게임 루프 통합 테스트: 건설→임대→수입→별점→NPC 반응
- macOS 네이티브 빌드 (맥미니 M4 Pro에서 직접 빌드/테스트)
- Windows 크로스 컴파일 + zip 패키징
- Ubuntu CI 통과 확인
